# Быстрый старт - Микросервисная архитектура

## Что изменилось?

Проект теперь разделен на:
- **Фронтенд** (`docs/`) - статические HTML/CSS/JS файлы
- **Бэкенд** (`api/`) - PHP API endpoints

## Как использовать

### 1. Локальный запуск (полная функциональность)

```bash
# 1. Убедитесь, что XAMPP запущен
# 2. Откройте браузер
http://localhost/CRM/docs/
```

Фронтенд автоматически подключится к PHP API на `http://localhost/CRM/api/`

### 2. Деплой на GitHub Pages (только интерфейс)

1. Перейдите в Settings → Pages вашего репозитория
2. Выберите папку `docs/` как источник
3. Сохраните

Через несколько минут фронтенд будет доступен на GitHub Pages.123
x
**Важно:** На GitHub Pages бэкенд недоступен, поэтому будет показано сообщение о необходимости локального запуска.

## Структура

```
docs/              ← Статический фронтенд (для GitHub Pages)
├── index.html
├── login.html
├── css/
└── js/
    ├── config.js  ← Автоопределение окружения
    ├── api.js     ← API клиент
    └── app.js     ← Утилиты

api/               ← PHP API (работает только локально)
├── cors.php       ← CORS настройки
├── search.php
├── company.php
└── ...
```

## Что работает где?

| Функция | GitHub Pages | Локально |
|---------|--------------|----------|
| Просмотр интерфейса | ✅ | ✅ |
| Вход/регистрация | ❌ | ✅ |
| CRUD операции | ❌ | ✅ |
| Поиск | ❌ | ✅ |
| Все функции CRM | ❌ | ✅ |

## Автоматическое определение

Файл `docs/js/config.js` автоматически определяет окружение:

- На GitHub Pages → показывает сообщение о необходимости локального запуска
- Локально → подключается к PHP API
