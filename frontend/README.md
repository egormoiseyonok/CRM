# Frontend для Simple CRM

Это статическая версия фронтенда, которая может быть задеплоена на GitHub Pages.

## Структура

```
frontend/
├── index.html          # Главная страница (Dashboard)
├── login.html          # Страница входа
├── css/                # Стили (скопированы из assets/css)
├── js/
│   ├── config.js       # Конфигурация (определение окружения)
│   ├── api.js          # API клиент
│   ├── app.js          # Основные утилиты
│   ├── layout.js       # Управление layout
│   └── pages/          # Скрипты для отдельных страниц
└── pages/              # Дополнительные HTML страницы
```

## Как это работает

### На GitHub Pages (статический хостинг)

- Фронтенд работает как статический сайт
- Показывается сообщение о необходимости локального запуска для полной функциональности
- Интерфейс доступен для просмотра

### Локально (с PHP бэкендом)

- Фронтенд автоматически определяет, что бэкенд доступен
- Все API запросы идут на локальный PHP сервер
- Полная функциональность доступна

## Деплой на GitHub Pages

1. Скопируйте папку `frontend/` в корень вашего репозитория (или в отдельную ветку `gh-pages`)
2. Настройте GitHub Pages в настройках репозитория
3. Укажите папку `frontend/` как корневую для GitHub Pages

### Альтернативный вариант

Если хотите использовать подпапку:

1. Переименуйте `frontend/` в `docs/`
2. В настройках GitHub Pages укажите папку `docs/`
3. Обновите пути в `config.js` если нужно

## Локальный запуск

Для полной функциональности:

1. Установите XAMPP или другой PHP сервер
2. Склонируйте репозиторий в `htdocs/CRM`
3. Настройте базу данных PostgreSQL
4. Откройте `http://localhost/CRM/` для PHP версии
5. Или откройте `http://localhost/CRM/frontend/` для фронтенда (он автоматически подключится к PHP API)

## Конфигурация

Файл `js/config.js` автоматически определяет окружение:

- Если хост содержит `github.io` или `github.com` → бэкенд недоступен
- Иначе → используется локальный бэкенд

Вы можете настроить URL бэкенда вручную в `config.js`, если нужно подключиться к удаленному серверу.
