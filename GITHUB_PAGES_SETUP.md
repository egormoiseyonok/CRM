# Настройка GitHub Pages для проекта CRM

## Быстрая настройка

1. **Перейдите в настройки репозитория:**
   - Откройте https://github.com/egormoiseyonok/CRM
   - Нажмите на вкладку **Settings**
   - В левом меню найдите раздел **Pages**

2. **Настройте источник:**
   - В разделе "Source" выберите **"Deploy from a branch"**
   - В поле "Branch" выберите вашу основную ветку (обычно `main` или `master`)
   - В поле **"Folder"** выберите **`/docs`** (это важно!)
   - Нажмите **Save**

3. **Дождитесь деплоя:**
   - GitHub автоматически задеплоит ваш сайт
   - Обычно это занимает 1-2 минуты
   - После деплоя вы увидите URL вашего сайта (например: `https://egormoiseyonok.github.io/CRM/`)

## Почему папка `docs/`?

GitHub Pages поддерживает только две опции:
- `/ (root)` - корень репозитория
- `/docs` - папка docs в корне репозитория

Поэтому мы переименовали `frontend/` в `docs/` для совместимости с GitHub Pages.

## Проверка деплоя

После настройки:

1. Подождите 1-2 минуты
2. Откройте URL вашего сайта (будет показан в настройках Pages)
3. Вы должны увидеть интерфейс CRM с сообщением о необходимости локального запуска

## Обновление сайта

После каждого push в репозиторий GitHub автоматически обновит сайт. Обычно это занимает 1-2 минуты.

## Локальная разработка

Для локальной разработки с полной функциональностью:

```
http://localhost/CRM/docs/
```

Фронтенд автоматически подключится к PHP API на `http://localhost/CRM/api/`

## Проблемы?

Если сайт не работает:

1. Убедитесь, что выбрана папка `/docs` (не `/`)
2. Проверьте, что в папке `docs/` есть файл `index.html`
3. Убедитесь, что изменения запушены в репозиторий
4. Подождите несколько минут - деплой может занять время
